# 모델 변경 정책 (Model Change Policy)

**문서 번호**: OPS-003
**버전**: 1.0
**최종 수정**: 2026-04-04

## 1. 목적

AI 분석 모델, 리스크 파라미터, 트레이딩 로직의 변경 시 안전한 배포와 검증 절차를 정의합니다.

## 2. 변경 대상 분류

| 분류 | 예시 | 승인 수준 | 검증 요건 |
|------|------|----------|----------|
| Critical | 리스크 게이트 임계값, 손실 한도 | 운영책임자 | 백테스트 + OOS + 페이퍼 1주 |
| High | Claude 프롬프트 변경, 스코어링 로직 | 개발 리드 | 백테스트 + OOS 검증 |
| Medium | 유니버스 필터 조건, 리밸런싱 주기 | 개발자 P2P | 단위 테스트 + 백테스트 |
| Low | 로그 포맷, 알림 문구 | 자동 승인 | 단위 테스트 |

## 3. 변경 절차

```
1. 변경 요청서 작성 (사유, 예상 영향, 롤백 계획)
   ↓
2. 백테스트 실행 (변경 전/후 비교)
   ↓
3. OOS (Out-of-Sample) 검증 (Critical/High만)
   ↓
4. 코드 리뷰 + 테스트 통과
   ↓
5. 승인 (분류에 따른 승인자)
   ↓
6. 스테이징 배포 + 페이퍼 트레이딩 (Critical만)
   ↓
7. 프로덕션 배포 (릴리스 게이트 통과)
   ↓
8. 모니터링 (변경 후 72시간 집중 관찰)
```

## 4. 백테스트 요건

- **기간**: 최소 2년 (2024-01-01 ~ 현재)
- **비교 지표**: 수익률, 샤프 비율, 최대 낙폭(MDD), 승률
- **허용 기준**: 기존 대비 MDD 악화 10% 이내, 샤프 비율 하락 0.2 이내
- **데이터**: 실제 시장 데이터 사용 (미래 데이터 참조 금지)

## 5. OOS 검증 요건

- 백테스트 미사용 구간에서 별도 검증
- 최소 6개월 미사용 데이터
- 백테스트 성과의 70% 이상 유지 시 통과

## 6. 롤백 절차

```
1. 이상 징후 감지 (성과 급락, 에러 증가)
   ↓
2. 매매 중단 (trading-halt-policy 적용)
   ↓
3. 이전 버전 배포 (Git tag 기반)
   ↓
4. 데이터 정합성 확인
   ↓
5. 매매 재개
   ↓
6. 변경 실패 사후 분석
```

## 7. 코드 연동 포인트

- `core/pipeline.py`: 게이트 임계값 (GateRegistry)
- `core/ai_analyzer/sentiment.py`: Claude 뉴스 센티먼트 분석 프롬프트
- `core/ai_analyzer/opinion.py`: Claude 종합 의견 스코어링 로직
- `core/ai_analyzer/prompt_manager.py`: 프롬프트 템플릿 관리
- `core/portfolio_manager/universe.py`: 유니버스 필터 조건
- `core/trading_guard.py`: 리스크 파라미터, 트레이딩 가드 로직
- `config/settings.py`: 모든 임계값의 환경변수 설정
- `config/operational_thresholds.yaml`: 운영 임계값 설정 (게이트, 리스크, 모니터링)
