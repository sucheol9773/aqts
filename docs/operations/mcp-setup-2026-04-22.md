# AQTS MCP Setup — 2026-04-22

> **문서 번호**: OPS-024
>
> **목적**: `.mcp.json` (Phase 4 migration 산출물) 의 per-developer 토큰 설정, Pilot worktree 격리, 운영 주의점 기록.
>
> **상위 migration**: `docs/operations/claude-multisession-migration-2026-04-22.md` (OPS-023) §5 Phase 4.

---

## 1. 개요

AQTS `.mcp.json` 은 프로젝트 공통 **opt-in MCP 서버 목록** 을 정의합니다. Claude Code 는 `.mcp.json` 이 존재하는 워크트리에서 `claude` 세션을 최초 기동할 때 **사용자에게 각 서버 approval 을 프롬프트** 하며, 승인하지 않으면 해당 세션에서는 MCP 가 활성화되지 않습니다. 즉 파일이 tracked 로 존재해도 자동 주입은 발생하지 않아 Pilot 관찰 데이터 오염 리스크는 최소화됩니다.

현재 Phase 4 에서 등록된 서버: **github (1 개)**.

---

## 2. 서버별 설정

### 2.1 github (`@modelcontextprotocol/server-github`)

**용도**: PR 관리, 이슈 트래킹, 코드 리뷰 코멘트. 기존 `gh` CLI 보다 구조화된 상호작용 (diff hunk 단위 리뷰 제안, issue linking) 가능.

**의존**: `npx` (Node.js 18+) — macOS / Linux 모두 표준. AQTS 는 이미 `.nvmrc` 없이 system Node 를 사용하므로 추가 설치 불필요.

**토큰 설정** (per-developer, untracked):

```bash
# ~/.zshrc 또는 ~/.bashrc 에 추가 (팀메이트 본인 PAT)
export GITHUB_PERSONAL_ACCESS_TOKEN="ghp_xxxxx..."
```

**필요 scope** (최소):
- `repo` — private repo 접근 (AQTS 는 public 이나 이슈/PR 본문은 때때로 민감)
- `read:org` — 조직 리뷰어 조회
- `workflow` (선택) — Actions 실행 이력 조회

**토큰 미설정 시 동작**: MCP 서버는 기동되나 GitHub API 호출 시 401. Claude 는 도구 호출 실패로 reported. 세션은 정상 진행.

### 2.2 GCP MCP (미포함, 검토 후 연기)

AQTS 는 현재 GCP 단일 VM 에 SSH 로 배포하며, 배포 orchestration 은 `.github/workflows/cd.yml` 이 전담합니다. 일상 운영 중 GCP API 직접 조회 필요는 낮아 Phase 4 범위에서 제외. 향후 staging VM 도입 / 다중 인스턴스 전환 시 재검토 (예상: ADR-004 이후).

## 3. Pilot worktree 격리 (ADR-002 Stage 2 관찰 중)

`aqts-team4-skills-pilot` worktree (브랜치 `pilot/team4-skills-w1`) 는 2026-04-22 ~ 2026-05-06 ADR-002 Stage 2 Sandbox 관찰 기간 중입니다. MCP 서버의 시스템 프롬프트 주입 / 도구 목록 확장이 skill trigger 정확도 평가를 오염시킬 수 있어 **MCP 전량 비활성** 필요:

```bash
# Pilot 리드가 aqts-team4-skills-pilot 에서 직접 수행
cat > .claude/settings.local.json <<'EOF'
{
  "disabledMcpjsonServers": ["*"]
}
EOF
```

`.claude/settings.local.json` 은 `.gitignore` 에 포함되어 untracked 로 유지됩니다. Stage 2 Exit 판정 (2026-05-06) 이후 Pilot MCP 재활성화 여부를 ADR-002 후속에서 별도 평가.

## 5. 운영 주의

### 5.1 토큰 회전

PAT 는 GitHub 설정의 **최대 1년** 만료로 생성. 만료 임박 시 `.zshrc` 업데이트. 일괄 관리 필요 시 향후 `op`/`gopass` 등 secret manager 연동 검토 (현재 범위 밖).

### 5.2 MCP 서버 에러 silent miss 방지 (development-policies.md §8)

Claude 세션 시작 시 `claude --debug` 또는 첫 응답의 MCP approval dialog 에서 서버 기동 여부를 확인. `npx` 가 proxy/network 차단 환경에서 실패하면 MCP 가 조용히 미로드된 채 세션이 진행될 수 있음.

검증 명령 (세션 기동 후):
- Claude 에게 `list MCP servers` 요청 → 승인된 서버 이름이 응답에 포함되는지 확인
- GitHub MCP 가 활성이면 `gh pr view <N>` 대신 `mcp__github__*` 도구 사용 가능

### 5.3 `.mcp.json` 스키마 검증

프로젝트 `.mcp.json` 은 Claude Code 공식 스키마 `https://json.schemastore.org/claude-code-mcp.json` 에 준하지만 현재 미선언. 향후 `wiring_smoke.sh` 에 `.mcp.json` 유효성 검사 추가 여부 검토.
