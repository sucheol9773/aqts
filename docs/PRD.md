# AQTS Product Requirements Document (1-Page PRD)

## 제품 개요

**제품명**: AQTS (AI Quant Trade System)
**버전**: 0.1.0-beta
**최종 수정**: 2026-04-04
**작성자**: AQTS 프로젝트팀

## 목적

AI 기반 퀀트 분석 + 자동매매 시스템을 통해 개인 투자자가 데이터 기반 투자 의사결정을 수행할 수 있도록 지원합니다. 초기 버전은 **단일 사용자 자가매매** 구조로 운영하며, 규제 요건 충족 후 자문형 서비스로 확장합니다.

## 서비스 형태

| 구분 | Phase 1 (현재) | Phase 2 | Phase 3 |
|------|---------------|---------|---------|
| 형태 | 자가매매 도구 | 자문형 (추천) | 일임형 (자동집행) |
| 고객 | 본인 | 개인 투자자 | 고액자산가/법인 |
| 규제 | 해당 없음 | 투자자문업 등록 | 투자일임업 등록 |
| 책임 | 사용자 전적 | 적합성 의무 | 선관주의 의무 |

## 핵심 기능

1. **AI 종목 분석**: Claude API 기반 펀더멘털/기술적 분석, 뉴스 센티먼트 통합
2. **포트폴리오 구성**: 리스크 프로필 기반 자동 유니버스 필터링 + 비중 최적화
3. **주문 실행**: KIS OpenAPI 연동, 한국/미국 시장 실시간 매매
4. **리스크 관리**: 9개 게이트 파이프라인, 손실 한도, 자동 중단
5. **모니터링**: 텔레그램 알림, 감사 로그, 성과 추적

## 기술 스택

FastAPI + PostgreSQL/TimescaleDB + MongoDB + Redis + KIS OpenAPI + Anthropic Claude

## 성공 지표

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 시스템 가용성 | 99.5% (장중) | Prometheus + AlertManager |
| 주문 실행 성공률 | 99% | orders 테이블 성공/실패 비율 |
| 분석 파이프라인 완료율 | 95% | pipeline 로그 COMPLETED/ERROR 비율 |
| API 응답 시간 (p95) | < 500ms | Prometheus histogram |

## SLA 정의

| 등급 | 대상 | 복구 목표 (RTO) | 데이터 복구 (RPO) |
|------|------|---------------|----------------|
| Critical | 주문 실행, 리스크 게이트 | 5분 | 0 (트랜잭션 무손실) |
| High | 환율 조회, 시세 수집 | 15분 | 5분 |
| Medium | 분석 파이프라인, 리포트 | 1시간 | 1시간 |
| Low | 감사 시각화, 대시보드 | 4시간 | 24시간 |

## 책임 경계

- **시스템 책임**: 주문 전송 실패, 데이터 수집 오류, 게이트 미작동 → 즉시 알림 + 자동 중단
- **시장 위험**: 가격 변동, 슬리피지, 시장 충격 → 사용자 인지 하에 실행, 시스템 면책
- **모델 위험**: AI 분석 오류, 편향 → 백테스트/OOS 검증 의무, 모델 변경 승인 절차
