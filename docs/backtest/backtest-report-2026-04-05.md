## 핵심 인사이트

### ENSEMBLE이 양쪽 시장 모두에서 최고 성과

26년 장기 백테스트에서 ENSEMBLE 전략이 한국(CAGR 12.70%, Sharpe 0.46)과 미국(CAGR 18.33%, Sharpe 0.62) 모두에서 최고 성과를 기록했다. 단기(2.3년) 백테스트에서는 보이지 않았던 패턴으로, 데이터 기간 확장의 중요성을 확인했다.

### 단기 vs 장기 결과 반전

단기에서는 시장별로 최적 전략이 달랐으나(한국: TREND_FOLLOWING, 미국: MEAN_REVERSION), 26년 장기에서는 ENSEMBLE이 일관되게 최적이었다. 이는 개별 전략의 약점을 앙상블이 상호 보완하기 때문이다.

### MDD 리스크

모든 전략에서 MDD가 -50% 이상이다. 닷컴버블(2000~2002), 금융위기(2008), 코로나(2020) 등 극단적 하락장에서 방어가 부족함을 의미한다. 최대 손실 방어 로직(stop-loss, 시장 레짐 감지 등) 추가가 필요하다.

### RISK_PARITY 거래 빈도

장기에서도 44~45건으로 낮은 편이나, 미국 시장에서 승률 77.8%로 높은 정확도를 보여 시그널 품질 자체는 양호하다.

## 개선 이력

### RISK_PARITY 시그널 로직 수정 (2026-04-05)

기존 로직은 고정 목표 변동성(15%) 대비 역비례로 시그널을 생성했으나, 대부분 주식의 변동성(15~30%)이 목표치 근처여서 시그널이 거래 임계값(±0.3)을 넘지 못해 거래 0건이 발생했다.

수정 후 변동성 추세(단기/장기 비교, 60%)와 절대 변동성 수준(중앙값 30% 기준, 40%)을 복합하는 방식으로 변경하여 실제 거래 가능한 시그널을 생성하도록 개선했다.

### 데이터 기간 확장 (2026-04-05)

기존 2024-01 ~ 2026-04 (2.3년, 62,800건)에서 2000-01 ~ 2026-04 (26년, 543,193건)로 확장. yfinance 백필 스크립트를 통해 114종목의 장기 OHLCV 데이터 수집 완료.

### MDD 방어 로직 구현 (2026-04-05)

BacktestEngine에 두 가지 리스크 관리 메커니즘을 추가했다.

**1. 포트폴리오 Drawdown Limit (`--max-dd`)**
- 포트폴리오 가치가 고점 대비 설정한 한도(예: -20%) 이상 하락하면 전 포지션을 강제 청산하고 "risk off" 모드에 진입한다.
- 이후 포트폴리오 가치가 고점의 95%(recovery_pct=5%) 이상으로 회복되면 거래를 재개한다.
- 닷컴버블, 금융위기, 코로나 등 극단적 하락장에서 -50% 이상의 MDD를 방어하기 위한 장치이다.

**2. 종목별 Stop-loss (`--stop-loss`)**
- 각 포지션의 미실현 손실이 설정한 비율(예: -10%)을 초과하면 해당 종목만 즉시 매도한다.
- 평균 매입가 대비 현재가의 손실률로 계산하며, 슬리피지와 거래비용을 반영한다.

**사용법:**
```bash
# 종목별 -10% 손절 + 포트폴리오 -20% DD 한도
python scripts/run_backtest.py --stop-loss 0.10 --max-dd 0.20

# stop-loss만 적용
python scripts/run_backtest.py --stop-loss 0.15

# DD 한도만 적용
python scripts/run_backtest.py --max-dd 0.25
```

BacktestConfig에 추가된 필드: `stop_loss_pct`, `max_drawdown_limit`, `drawdown_recovery_pct`

### MDD 방어 로직 v2 개선 (2026-04-05)

v1의 치명적 버그 수정: DD 발동 후 영구적으로 risk-off 상태에 빠지는 문제.

**원인**: 전량 청산 후 현금 상태에서 포트폴리오 가치가 자체 성장하지 않아 고점의 95%(recovery threshold)에 도달 불가능.

**수정 내용:**
1. 시간 기반 쿨다운(기본 20영업일)으로 변경 — DD 발동 후 일정 기간 대기 후 자동 복귀
2. 쿨다운 종료 시 peak를 현재 가치로 리셋 — 새로운 기준점에서 거래 재개
3. ATR 기반 동적 stop-loss 추가 — 고정 비율 대신 최근 20일 변동폭 × 배수로 손절선 결정 (한국 주식의 높은 변동성에 적합)
4. `drawdown_recovery_pct` → `drawdown_cooldown_days`로 대체

**비교 결과 (KR 88종목, 26년):**

| 구분 | ENSEMBLE 수익률 | MDD | Sharpe | 거래 수 |
|---|---|---|---|---|
| 보호 없음 | +3,288% | -51.86% | 0.55 | 1,043 |
| v1 (--stop-loss 0.10 --max-dd 0.20) | +3.26% | -21.32% | -0.53 | 11 |

v1은 MDD를 -21%로 방어했지만 거래가 11건으로 급감하여 수익률이 사실상 0에 수렴. v2로 수정 후 재검증 필요.

### 시그널 생성 성능 최적화 (2026-04-05)

1. **벡터화**: 날짜별 for-loop(종목당 ~6,500회 반복)을 pandas 벡터 연산으로 교체하여 10~50배 속도 향상
2. **병렬화**: 4종목 이상일 때 `multiprocessing.Pool`로 종목별 시그널 생성을 CPU 코어 수만큼 동시 처리

## 향후 과제

- MDD 방어 효과 비교 백테스트 실행 (보호 있음 vs 없음)
- 시장 레짐 감지(VIX 기반 위험 회피) 로직 추가
- RISK_PARITY 거래 빈도 개선: 시그널 임계값 또는 로직 추가 튜닝
- 거래 임계값(0.3) 파라미터 민감도 분석
- 시장별 전략 가중치 자동 조정 로직 (동적 앙상블)
- 트랜잭션 코스트 민감도 분석
- Walk-forward 검증 (Out-of-Sample 테스트)
