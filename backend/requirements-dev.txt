# ══════════════════════════════════════
# AQTS Backend — Dev / CI 전용 의존성
# ══════════════════════════════════════
#
# 목적:
#   - 운영 런타임이 아닌 **개발/CI 전용** 도구 (lint, format) 를 분리 관리.
#   - CI 의 lint 스텝은 본 파일을 단일 진입점으로 사용하여 `.github/workflows/`
#     에 버전 번호가 하드코딩되는 패턴을 제거한다 (CLAUDE.md §1 "하드코딩 금지").
#
# 로컬 개발자 셋업:
#   런타임과 dev 도구를 모두 설치하려면 두 파일을 순서대로 넘긴다.
#   `pip install -r backend/requirements.txt -r backend/requirements-dev.txt`
#
#   lint 도구만 필요한 경우 (예: editor/LSP 셋업):
#   `pip install -r backend/requirements-dev.txt`
#
# 설계 결정 — `-r requirements.txt` 를 포함하지 않는 이유:
#   본 파일이 runtime 의존성을 포함하면 CI 의 lint 잡(ruff/black check 만 수행)
#   이 FastAPI/SQLAlchemy/Motor 등 수십 MB 의 런타임 패키지까지 매번 설치하게
#   된다. lint 는 import 를 실행하지 않으므로 불필요. 로컬 개발자는 위 두
#   파일 명령으로 동일한 편의를 얻는다.
#
# 버전 업그레이드 절차:
#   1. 본 파일에서 버전만 bump 하고 로컬에서 `ruff check` + `black --check`
#      실행 → 규칙 drift 관찰.
#   2. CI 가 본 파일을 자동으로 설치하므로 `.github/workflows/ci.yml` 수정 불필요.
#   3. 커밋 메시지에 "before/after lint diff 줄 수" 를 포함한다.
#
# 제외 대상 (의도적으로 본 파일에 넣지 않음):
#   - pytest* : `requirements.txt` 에 고정되어 있어 smoke/test 잡이 그대로 참조.
#     별도 분리는 `Dockerfile` 의 `-r requirements.txt` 범위를 건드리는 더 큰 변경.
#   - pip-audit : 용도가 "security scan" 이라 `backend/requirements-security.txt`
#     로 분리됨 (2026-04-25, OPS-025 — `docs/operations/security-deps-split-2026-04-25.md`).
